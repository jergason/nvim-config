# CLAUDE.md

## What This Is

neovim configuration using fennel (a lisp that compiles to lua) via nfnl. the config lives in `.config/nvim/` and gets symlinked to `$HOME/.config/nvim`.

## Architecture

**bootstrap flow:** `init.lua` → bootstraps lazy.nvim + nfnl → loads `.config/nvim/fnl/config/init.fnl`

## Plugin Management

We use lazy.nvim to manage plugins. Plugins are in loaded from `.config/nvim/fnl/config/plugin.fnl`. Most plugins reference a module in `.config/nvim/fnl/config/plugin/` for their configuration if they need more complicated setup or it can't happen inline. LSP-specific configs are in `.config/nvim/fnl/config/plugin/lsp/`.

**key directories:**

- `.config/nvim/fnl/config/` - fennel source files (authoritative)
- `.config/nvim/fnl/config/plugin/` - per-plugin configuration modules
- `.config/nvim/fnl/config/plugin/lsp/` - lsp-specific configs
- `.config/nvim/fnl/ftplugin/` - filetype-specific settings
- `.config/nvim/lua/` - compiled lua output (generated by nfnl, don't edit directly)

**compilation:** nfnl compiles `.fnl` → `.lua` on save. the `.nfnl.fnl` config file is empty (`{}`), using defaults.

## Working with Fennel

fennel uses s-expressions. quick reference:

- `(fn name [args] body)` - define function
- `(local x value)` - local binding
- `(set vim.g.foo "bar")` - assignment
- `(tset table key value)` - table set
- `{:key value}` - table literal (`:key` is a string)
- `#(body)` - anonymous function shorthand
- `(require :module.name)` - require (`:` prefix = string)

## Key Mappings

mappings defined in `fnl/config/init.fnl` and per-plugin configs.

`config.util` module provides `nnoremap`/`vnoremap` helpers that auto-prefix leader and wrap in `<cmd>`.
